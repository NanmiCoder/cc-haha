# v0.5.18

> 技能系统升级：内置 Karpathy Guidelines、激活机制自动注入、CLAUDE.md 管理 UI。

## 新功能

- **Karpathy Guidelines 技能内置到 Catalog**（#70）。Settings → Skills 推荐列表新增 "Karpathy Guidelines" 技能，一键安装即可使用。涵盖 4 大原则：Think Before Coding、Simplicity First、Surgical Changes、Goal-Driven Execution。来源：multica-ai/andrej-karpathy-skills（MIT）。

- **技能激活机制（Active Skills）**（#71）。支持将已安装技能标记为"始终激活"，分为全局（`~/.claude.json` 中 `activeSkills`）和项目级（`.claude/settings.json` 中 `activeSkills`）两个作用域。激活后的技能 SKILL.md 内容在每次对话开始时自动注入 system prompt，无需手动 `/slash-command` 调用。项目级优先，重复技能自动去重。
  - 新增 `GET /api/skills/active?scope=global|project|merged` 查询接口
  - 新增 `POST /api/skills/active` 设置接口（支持 scope + skills 参数）
  - Desktop API：`skillsApi.getActiveSkills()` / `skillsApi.setActiveSkills()`

- **CLAUDE.md 项目规则管理 UI**（#72）。Settings 新增 "Project Rules" 标签页，可视化管理项目级（`.claude/CLAUDE.md`）和用户级（`~/.claude/CLAUDE.md`）规则文件。支持"打开"已有文件或"创建"模板文件。
  - 新增 `GET /api/project-rules` 查询文件路径及存在状态
  - 新增 `POST /api/project-rules/create` 创建模板文件
  - 全部 5 语言 i18n 支持（en/zh/zh-TW/jp/kr）
