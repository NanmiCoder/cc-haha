# v0.5.17

> MCP 设置页连接可靠性大幅提升 + 配置编辑快捷入口。

## 新功能

- **MCP 设置页"编辑 JSON"按钮**（#69）。MCP 列表页顶部新增"编辑 JSON"按钮，点击直接用系统默认编辑器打开 `~/.claude.json` 配置文件，无需再通过繁琐的表单逐项填写。新增 `GET /api/mcp/config-files` 端点返回各 scope 的配置文件路径。
- **MCP 每服务器手动刷新按钮**（#69）。每个 MCP server 行旁新增刷新图标按钮，可随时手动重新检测连接状态，无需关闭重开设置页。

## 修复

- **MCP 连接超时从 30s 增至 60s**（#69）。`npx -y` / `uvx` 命令首次运行需要下载包，30 秒经常超时显示 Unavailable。增至 60 秒后覆盖大多数首次下载场景。环境变量 `MCP_TIMEOUT` 仍可自定义。
- **MCP 状态检查全并发**（#69）。前端从 2-worker 串行队列改为全并发 `Promise.all`，打开设置页后所有 server 同时探测，整体耗时等于最慢的那一个。
- **MCP 状态服务端缓存**（#69）。服务端新增 5 分钟 TTL 的状态缓存，重复打开设置页时直接返回上次已知状态（connected/failed），不再每次重新连接。reconnect/toggle 操作自动清除缓存保证时效性。
- **MCP 超时自动重试**（#69）。首次探测因超时失败后，5 秒延迟自动重试一次（此时包通常已部分缓存），无需用户手动操作。

## 范围

- `src/services/mcp/client.ts`：`getConnectionTimeoutMs()` 默认值 30000→60000。
- `src/server/api/mcp.ts`：新增 statusCache（Map + TTL）、`clearStatusCache()` 导出、`getConfigFilePaths()` 端点、`serializeServerSnapshot` 使用缓存。

- `desktop/src/api/mcp.ts`：新增 `configFiles()` 方法。
- `desktop/src/i18n/locales/{en,zh,zh-TW,jp,kr}.ts`：新增 `settings.mcp.editConfig` / `editConfigTooltip` 翻译。
- 测试：`src/server/__tests__/mcp-status-cache.test.ts`（2 cases）、`desktop/src/__tests__/mcpRefreshButton.test.tsx`（2 cases）。
