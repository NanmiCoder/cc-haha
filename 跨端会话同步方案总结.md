## ✅ 问题二：Android 与 Windows 会话同步方案

### 方案架构图：

```
┌─────────────────────────────────────────────────────────────────────┐
│                          本地电脑 (Windows/Mac/Linux)               │
│                                                                      │
│  ┌──────────────────┐         ┌───────────────────────────────┐    │
│  │  Tauri 桌面端    │         │  Bun 本地服务器 (0.0.0.0:3456) │    │
│  │  (React + Rust)  │ ──────► │                               │    │
│  └──────────────────┘         │  - REST API (/api/sessions)   │    │
│                               │  - WebSocket (/ws/:sessionId) │    │
│                               └──────────────┬────────────────┘    │
│                                              │                     │
│                                              │ 读写                 │
│                                              ▼                     │
│                               ┌───────────────────────────────┐   │
│                               │  ~/.claude/projects/          │   │
│                               │  [sanitized-path]/            │   │
│                               │  [session-id].jsonl           │   │
│                               └───────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
                                      │
                                      │ 同一局域网
                                      │ HTTP/WebSocket
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────┐
│                          Android 手机                                │
│                                                                      │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │  React Native 客户端 (Expo)                                  │  │
│  │                                                              │  │
│  │  - SessionListScreen: 查看/选择会话                          │  │
│  │  - ChatScreen: 实时对话                                      │  │
│  │  - ServerConfigScreen: 配置服务器地址                        │  │
│  │                                                              │  │
│  │  使用 Zustand 状态管理                                       │  │
│  │  复用现有 API 结构                                           │  │
│  └──────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────┘
```

### 关键同步机制：

1. **共享数据源**：所有端读写同一份 JSONL 文件
2. **实时通信**：WebSocket 推送消息更新
3. **API 一致**：Android 复用桌面端的 API 接口设计

## 🚀 快速开始使用

### 第一步：启动本地服务器（电脑端）

Windows 用户：
```cmd
# 双击运行
start-server.bat
```

或手动：
```powershell
$env:SERVER_HOST="0.0.0.0"
$env:SERVER_PORT="3456"
bun run src/server/index.ts
```

### 第二步：获取电脑 IP 地址

```cmd
ipconfig
# 找到类似 "192.168.1.100" 的地址
```

### 第三步：安装运行 Android 端

```bash
cd android
bun install
bun start
```

在手机上：
1. 安装 Expo Go App
2. 扫描终端二维码
3. 进入 "Server Config" 输入 `http://192.168.1.100:3456`（替换为你的 IP）
4. 点击 "Test Connection" 测试
5. 开始使用！

## 💡 核心设计亮点

### 1. 完全复用现有架构
- API 接口与桌面端一致
- 会话存储格式（JSONL）兼容
- WebSocket 协议复用

### 2. 本地优先，无云依赖
- 数据全部存储在用户本地电脑
- 不需要外网服务器
- 隐私安全有保障

### 3. 易于扩展
- 未来可以添加 iOS 端
- 可以添加更多 IM 适配器
- 架构支持插件化扩展
